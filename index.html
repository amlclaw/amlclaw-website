aml-rule-generator

AML Rule Generator

Reads global regulatory documents — FATF 40 Recommendations, MAS DPT Notices, VARA frameworks — and synthesizes localized, machine-readable rules.json policies.

  • Singapore MAS / Hong Kong SFC / Dubai VARA
  • Zero-tolerance CFT enforcement logic
  • Tier-based risk actions (Freeze / EDD / Review)

Address Screening KYA

Deep-scans a blockchain wallet up to 5 hops, mathematically maps fund flows, evaluates exposure against your rules.json, and outputs a professional Markdown Audit Report.

  • 5-hop bidirectional graph traversal
  • Automated evidence tracing (A ——$——> B)
  • LLM-generated compliance reports

Architecture

Designed for AI agents.
Not just humans.

01

Standard SKILL.md Interface

Each module exposes a SKILL.md file that acts as an AI-readable instruction card. Your LLM agent automatically discovers what the skill does and how to invoke it.

02

Python Data Pipeline

Lightweight Python scripts handle the heavy lifting — fetching blockchain graphs via API, extracting risk-relevant paths, and compiling the data into a context-optimized JSON file.

03

LLM as the Evaluation Engine

The LLM agent reads the condensed risk data and your rules.json, applies rule logic internally, and generates a structured compliance report — no backend server needed.

04

Zero Vendor Lock-in

Works with Claude Code, OpenClaw, Cursor, or any filesystem-aware LLM agent. Your rules are a local JSON file — you own your compliance policy completely.

Installation

Up and running in
under two minutes.

1

Clone into your workspace

bash 
git clone https://github.com/amlclaw/amlclaw.git ./skills/amlclaw
2

Install Python dependencies

bash 
pip install -r skills/amlclaw/aml-address-screening/requirements.txt
3

Configure your API key

bash 
cp skills/amlclaw/.env.example .env
# Edit .env and add your TRUSTIN_API_KEY
4

Start your agent and prompt it

prompt 
# Example agent prompt:
Use the AML Address Screening skill to investigate Tron address 
TGE94jU39ithtHbrYAQJRTcvv785riPLdy against our rules.json. 
Check inflows AND outflows up to 5 hops, then produce a 
full compliance report.
