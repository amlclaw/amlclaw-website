V2 Features

Smarter screening.
Deeper intelligence.

Version 0.2.0 introduces four major capabilities for precision compliance.

Scenario-Based Screening

6 screening scenarios automatically filter rules by business context and direction. Each scenario targets specific risk categories.

onboarding deposit withdrawal cdd monitoring all
--scenario deposit --inflow-hops 5

Pollution Decay Principle

Risk severity attenuates with hop distance. Close-range exposure triggers maximum severity; distant connections reduce to proportional risk.

Hop 1
Severe
Hop 2–3
Severe
Hop 4–5
High

Self-Tag Detection

Evaluates the target address's own blockchain labels — sanctions, darknet, gambling — independent of path traversal. Critical for KYC onboarding.

"parameter": "target.tags.primary_category" "operator": "equals" "value": "Sanctioned Entity"

Direction-Aware Rules

Rules independently target inflow (fund sources) or outflow (fund destinations). Withdrawal rules only scan outgoing paths; deposit rules focus on incoming.

inflow
outflow
"direction": "inflow" | "outflow"
Agent Skills

Two powerful modules.
One seamless pipeline.

Install once. Your AI inherits enterprise compliance capabilities instantly.

aml-rule-generator

AML Rule Generator

Reads global regulatory documents — FATF 40 Recommendations, MAS DPT Notices, VARA frameworks — and synthesizes localized, machine-readable rules.json policies.

  • 4 business scenario categories (Deposit, Withdrawal, CDD, Monitoring)
  • Hop-tiered severity rules (Pollution Decay Principle)
  • Pre-built rulesets: SG (19), HK (18), Dubai (19) rules
  • JSON Schema validation with TrustIn label cross-check
./aml-rule-generator/
aml-address-screening

Address Screening KYA

Deep-scans a blockchain wallet up to 5 hops, mathematically maps fund flows, evaluates exposure against your rules.json, and outputs a professional Markdown Audit Report.

  • 6 screening scenarios (onboarding, deposit, withdrawal, cdd, monitoring, all)
  • Direction + hop-based rule filtering
  • Target self-tag detection (DEP-SELF-*, WDR-SELF-*)
  • Auto update notifications for skill users
./aml-address-screening/
Architecture

Designed for AI agents.
Not just humans.

01

Standard SKILL.md Interface

Each module exposes a SKILL.md file that acts as an AI-readable instruction card. Your LLM agent automatically discovers what the skill does and how to invoke it.

02

Python Data Pipeline

Lightweight Python scripts handle the heavy lifting — fetching blockchain graphs via API, scenario-based filtering of risk paths, and direction-aware evidence extraction with hop-tiered severity.

03

LLM as the Evaluation Engine

The LLM agent reads condensed risk data and your rules.json, applies hop-tiered severity reasoning and self-tag evaluation, and generates structured compliance reports with evidence chains.

04

Zero Vendor Lock-in

Works with Claude Code, OpenClaw, Cursor, or any filesystem-aware LLM agent. Built-in version checker notifies when skill updates are available. Your rules are a local JSON file — you own your compliance policy.

Installation

Up and running in
under two minutes.

1

Clone into your workspace

bash 
git clone https://github.com/amlclaw/amlclaw.git ./skills/amlclaw
2

Install Python dependencies

bash 
pip install -r skills/amlclaw/aml-address-screening/requirements.txt
3

Configure your API key

bash 
cp skills/amlclaw/.env.example .env
# Edit .env and add your TRUSTIN_API_KEY
4

Start your agent and prompt it
